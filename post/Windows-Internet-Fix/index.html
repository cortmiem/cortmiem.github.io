修复 Windows 中的网络连接问题

2020-08-16 3 min read

学者沃·兹基·燕鸠德的报告指出：

电脑维修：重启解决90%问题，重装解决99%问题，重买解决100%问题。

一般方案

检查线路连接

在命令提示符中运行以¹获取网关、²测试网关连接：

ipconfig
ping 192.168.1.1

根据微软官方文件微软官方文件（有删改）：

如果你收到回复，则表明你已连接到 Wi-Fi 路由器，因此可能是你的调制解调器或 ISP 有问题。请联系你的 ISP，或者在另一台设备上联机检查（如果可以），看看是否存在服务中断。
如果 ping 测试的结果指出你未收到路由器的回复，请尝试使用以太网电缆将你的电脑直接连接到调制解调器（如果可以）。如果你可以使用以太网电缆连接到 Internet，它会确认连接问题是由 Wi-Fi 路由器引起的。请确保你已安装最新固件，并查看路由器的相关文档。

运行网络命令

在命令提示符中运行以¹手动重置 TCP/IP 堆栈、²释放并续订 IP 地址，³⁴⁵刷新并重置 DNS 客户端解析程序缓存：

netsh winsock reset
netsh int ip reset
ipconfig /release
ipconfig /renew
ipconfig /flushdns

检查操作系统

使用其他操作系统或环境进行网络连接（如：Porteus、CDLinux、带网络连接Windows PE）

使用三方软件

以上步骤可部分使用三方软件代替。

  • 360、电脑管家等（不推荐）；
  • ComIntRep（Complete Internet Repair）
    该软件可以¹重置 Internet 协议（TCP/ IP 协议）、²修复 Winsock、³更新互联网连接、⁴刷新 DNS 解析器缓存、⁵刷新 ARP 缓存、⁶修复 Internet Explorer、⁷清除 Windows 更新历史记录、⁸修复 Windows 自动更新、⁹修复的 SSL / HTTPS 加密、⁰重置代理服务器配置、¹¹重置 Windows 防火墙配置、¹²恢复默认的 hosts 文件、¹³修复查看工作组计算机

特殊情况

使用代理进行科学上网后使用代理进行科学上网后（此计算机上缺少一个或者多个网络协议）：

  • 重新启动代理服务应用程序（如 Shadowsocksr）；
  • 关闭代理设置脚本、停用代理服务器：
  • 位于“Windows设置 - 网络和Internet - 代理”；
  • 删除注册表中的连接信息（删除 Connections 项下除默认以外的所有值）。
HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Internet Settings\Connections

使用WLAN连接的台式机用户：

  • 断开 RJ45 网络电缆线（网线）；
  • 停用本地连接或有线网络适配器：
    位于“控制面板 - 网络和 Internet - 网络和共享中心 - 更改适配器设置”。

使用笔记本电脑的用户：

请确保你的笔记本电脑上的物理 Wi‑Fi 开关已打开。（当它处于打开状态时，指示灯通常会亮起。）

使用 Fiddler 4 解除 UWP 应用的网络隔离：
位于“Fiddler 4 - winConfig - Exempt All - Save Changes”。
